Kiến trúc Monolithic ​

  • Là kiến trúc phía Server của ứng dụng dạng Client-Server
  • Toàn bộ team làm việc trên một Project(Solution)
  • Được đóng gói thành một file hoặc một thư mục khi triển khai
  • Dễ dàng develop, deploy, scale với những dự án nhỏ

Mở rộng hệ thống : ​

  • Vertical Scaling: chính là nâng cấp Ram, Cpu, SSD cho Server để có thể đáp ứng được khả năng mở rộng
  • Horizontal Scaling: Cái này thì sẽ phức tạp hơn , mở rộng về chiều ngang đòi hỏi phải cấu hình nhiều hơn:
    • Tạo nhiều Instance của Server giống hệt bản gốc ( ví dụ tạo thêm 2 bản nữa )
    • Lúc này không chỉ có một Server để phục vụ cho ứng dụng => Vấn đề: Làm thế nào để phân loại request, cache, ... cho từng Instance của Server.
    • Resolve: Load Balance
    • Tạo thêm một(or N) Database nữa , phân loại database dựa trên database nào đọc( Cấu hình cao hơn) , database nào để ghi, sau một thời gian cần đồng bộ dữ liệu từ Database ghi vào Database đọc

=> Vấn đề : ​

  • Chỉ phù với những dự án nhỏ, hoặc ngắn hạn => những dự án Outsource cần tiến độ nhanh.
  • Với những dự án lớn, nhiều người tham gia:
    • IDE bị quá tải, chậm.
    • Khó mở rộng, phân chia đội ngũ.
    • Thường gắn gắn chặt với một nền tảng, công nghệ, kém linh hoạt.
